超入門 図解

Next.js ってなに？

専門用語をわかりやすく解説しながら、Webサイトが表示される仕組みを図解します

そもそも Next.js ってどんなもの？

Next.js は Webサイト・Webアプリを作るためのフレームワーク（道具箱） です。
料理で言えば「包丁・まな板・鍋がセットになったプロ用キッチン」のようなもの。自分で1から揃えなくても、必要な道具がまとまって使えます。

🏗️

React（基礎）

画面を部品（コンポーネント）に分けて作る仕組み。Next.jsはこれをベースに使います。

📦

Next.js（道具箱）

Reactに「高速化」「SEO対策」「ページ管理」などの便利機能を追加した上位版。

🌐

完成するもの

ブログ・ECサイト・企業サイト・Webアプリ何でも作れます。

あなたがWebサイトを開いたとき、何が起きている？
🧑‍💻

① あなた（ユーザー）がURLを入力

ブラウザ（ChromeやSafari）に「https://example.com」と入力してEnterを押す

↓
🖥️
💡 用語：サーバー

② サーバーにリクエストが届く

サーバーとは「24時間動いているコンピュータ」のこと。インターネット上のどこかにあり、あなたのリクエストを受け取ります。

↓
⚙️
💡 用語：レンダリング

③ Next.js がHTMLを作る（レンダリング）

レンダリングとは「データやコードをブラウザで表示できるHTMLに変換する作業」のこと。料理で言えば「材料から料理を完成させる工程」です。

↓
📤
💡 用語：レスポンス

④ HTMLがあなたのブラウザに届く（レスポンス）

レスポンスとは「サーバーからの返事」のこと。リクエスト（質問）に対してレスポンス（回答）が返ってきます。

↓
🎉

⑤ ページが画面に表示される

ブラウザがHTMLを解釈して、画像・文字・ボタンなどが見えるようになります。

Next.js の「ページ作り方」3パターン

Next.js の一番の特徴は「いつ・どこでHTMLを作るか」を選べること。状況に応じて使い分けます。

🏭 事前に作っておく

SSG

Static Site Generation（静的サイト生成）

サービスを公開する前にあらかじめHTMLを全部作っておく方式。お客さんが来たらすぐ渡せるように「おにぎりを作り置きしておく」イメージ。

✅ ブログ、会社紹介ページ、変更が少ないページ向き

表示速度
🍳 その場で作る

SSR

Server Side Rendering（サーバーサイドレンダリング）

お客さんが来るたびにサーバーでHTMLを作って返す方式。「注文を受けてから料理を作る」レストランのイメージ。常に最新情報を表示できます。

✅ ログイン後のページ、リアルタイムデータを表示するページ向き

表示速度
💻 ブラウザで作る

CSR

Client Side Rendering（クライアントサイドレンダリング）

HTMLはほぼ空で送り、あなたのブラウザ（クライアント）がJavaScriptでページを組み立てる方式。「材料を送ってお客さん自身で料理してもらう」イメージ。

✅ ダッシュボード、インタラクティブなアプリ向き

表示速度
Next.js プロジェクトのフォルダ構成

Next.js を使い始めると以下のようなフォルダ・ファイルが自動で作られます。最初は全部理解しなくてOK！

📁 my-app/ ← プロジェクトのフォルダ
📁 app/ ページを置くメインフォルダ（最重要）
📄 page.tsx トップページ（/）の中身
📄 layout.tsx 全ページ共通の枠（ヘッダーなど）
📄 globals.css サイト全体のデザイン設定
📁 public/ 画像・アイコンなどを置く場所
📄 package.json 使っている道具（ライブラリ）の一覧表
📄 next.config.js Next.jsの細かい設定ファイル

💡 ポイント：app/ フォルダの中に about/page.tsx を作ると、自動で /about というURLのページになります！（ファイルベースルーティング）

よく出てくる専門用語まとめ

🔧 フレームワーク

開発を楽にするための「あらかじめ準備された道具とルールのセット」

👉 プロ用の調理器具セットみたいなもの

🧩 コンポーネント

画面をボタン・ヘッダーなど小さな「部品」に分けたもの。使い回しできる。

👉 レゴブロックの1ピース

📡 API

サービス同士がデータをやり取りする窓口。天気アプリが天気データを取得するときなどに使う。

👉 コンビニのイートインの注文カウンター

🗺️ ルーティング

URLと表示するページを対応させる仕組み。

👉 /about → 会社概要ページ、という道案内

📚 ライブラリ

よく使う処理をまとめた「便利な関数集」。npm で追加できる。

👉 料理本（使いたいときだけ開く）

🚀 デプロイ

作ったWebサイトをサーバーに置いて、世界中からアクセスできる状態にすること。

👉 お店のシャッターを開けること

まとめ

📌 Next.js を一言でいうと

  • 1 React をベースに作られた Webサイト・アプリ開発のプロ用道具箱
  • 2 SSG・SSR・CSR の3方式を状況に応じて選べるのが最大の強み
  • 3 ファイルを置くだけでURLが自動的に作られる（ファイルベースルーティング）
  • 4 Vercel 社が作っており、そのまま Vercel でデプロイ（公開）するのが最も簡単
