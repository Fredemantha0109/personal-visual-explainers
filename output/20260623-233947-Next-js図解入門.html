Reactをベースにしたウェブアプリ開発フレームワーク。
「速い・作りやすい・SEOに強い」を一気に実現できる道具です。

1Next.js の全体像

ユーザーがURLにアクセス → Next.jsサーバーがページをHTMLに変換してブラウザへ送る。必要ならDBやAPIからデータを取ってくる。

2「いつHTMLを作るか」の3パターン

⚠️ レンダリングとは「HTMLを作って画面に描く処理」のことです。「いつ・どこで」作るかで3種類に分かれます。

🏗️ ビルド時に事前に作る

SSG

Static Site Generation（静的生成）

サイトを公開する前に全ページのHTMLを作り置き。リクエストが来たら即座に返すだけ。

✅ 最速・サーバー負荷ゼロ
❌ 内容を更新しにくい

例）ブログ記事、製品説明ページ

🔄 リクエストごとに毎回作る

SSR

Server Side Rendering（サーバー側で描画）

ユーザーがアクセスするたびにサーバーがHTMLを生成して送る。

✅ 常に最新データを表示
❌ サーバーへの負荷が高い

例）ニュースサイト、ユーザー固有のページ

💻 ブラウザ側で動的に作る

CSR

Client Side Rendering（ブラウザ側で描画）

最初はほぼ空のHTMLを送り、JavaScriptでブラウザ上に描画。

✅ インタラクティブな操作が得意
❌ 初回表示が遅い・SEO弱い

例）管理画面、リアルタイムチャット

⏱ 速度イメージ比較

3ファイル名がそのままURLになる！

ルーティングとは「このURLを開いたらどのページを表示するか」の振り分けのこと。
Next.jsではapp/フォルダのファイル構造がそのままURLになります。設定ファイルは不要！

app/page.tsx → example.com/ （トップページ）
app/about/page.tsx → example.com/about （会社概要ページ）
app/blog/page.tsx → example.com/blog （ブログ一覧）
app/blog/[id]/page.tsx → example.com/blog/123 （記事ID=123の詳細）

💡 [id] のように [ ] で囲むと「どんな値でも受け取れる動的なURL」になります。

4Next.jsを使うメリット
⚡
表示が高速
事前にHTMLを用意する仕組みにより、ページの読み込みが爆速になる。
🔍
SEOに強い
検索エンジンが読めるHTMLを最初から返すので、Google検索の上位に載りやすい。
📁
設定がシンプル
ファイルを置くだけでURLが決まる。複雑な設定ファイルが要らない。
🎛️
柔軟に使い分け
SSG/SSR/CSRをページごとに選択できる。用途に応じた最適な方法をとれる。
🔒
API機能内蔵
サーバー側の処理（DB操作・認証など）も同じプロジェクト内に書ける。

📝 まとめ：Next.jsとは？

Reactで作ったUIを、SEOに強く・高速に・簡単にWebで公開できる仕組みです。
「ファイルを置けばURLが生まれ、用途に応じてサーバー/ブラウザどちらでも動かせる」
