APIは「お客さん(アプリ)」と「キッチン(サーバー)」の間で注文を取り次ぐウェイター役

② API通信の流れ（4ステップ）
1
アプリがリクエストを送る
「天気を教えて」「ユーザー情報をください」など、必要なデータや操作を要求する。
GET https://api.example.com/weather?city=tokyo
2
APIがリクエストを受け取る
「誰が」「何を」「どんな権限で」求めているかをチェック（認証・バリデーション）。
3
サーバーがデータを処理・返却
データベースを検索・計算して結果を用意する。
4
アプリがレスポンスを受け取る
主にJSON形式でデータが返り、アプリが画面に表示する。
{ "city": "tokyo", "temp": 28, "weather": "晴れ" }
③ REST API — 4つの基本操作
GET
データを取得
例）商品一覧を読み込む
POST
データを新規作成
例）新規ユーザーを登録する
PUT
データを更新
例）プロフィールを変更する
DELETE
データを削除
例）投稿を削除する

これら4つで「作る・読む・更新・削除」= CRUD 操作を表現する

⑤ APIのポイントまとめ
🔌
接続口の標準化：内部の複雑な仕組みを隠し、決まった形式でやり取りできるようにする
♻️
再利用性：一度作ったAPIは複数のアプリ（Web・スマホ等）から使い回せる
🔒
セキュリティ：APIキーやトークンで「許可された人だけ」がアクセスできるよう制御する
⚡
開発効率：地図・決済・AIなど高機能をゼロから作らずすぐに使える
