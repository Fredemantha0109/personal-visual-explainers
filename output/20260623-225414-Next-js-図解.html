▍ Overview

Next.js とは？

Vercel 社が開発した React ベースのフルスタックフレームワーク。 複数のレンダリング戦略・ファイルベースルーティング・API Routes などを標準装備し、 パフォーマンスと開発体験を両立する。

⚛️ React 🚀 Vercel 📦 Node.js 🌐 Full-Stack 📁 File-based Routing
▍ Rendering Strategies（レンダリング戦略）
🏗️
SSG
Static Site Generation
ビルド時にHTMLを生成。
CDN配信で超高速。
変更が少ないコンテンツ向け。
例: ブログ・LP・ドキュメント
🔄
SSR
Server-Side Rendering
リクエスト毎にサーバーでHTML生成。
常に最新データを提供。
SEOにも強い。
例: ECサイト・ダッシュボード
⏱️
ISR
Incremental Static Regeneration
一定時間後にバックグラウンドで再生成。
SSGの速さ＋SSRの新鮮さ。
例: ニュースサイト・商品一覧
💻
CSR
Client-Side Rendering
ブラウザ側でJSがレンダリング。
useEffect / SWR で動的取得。
例: 認証後の個人ページ
速度 vs 新鮮さ のトレードオフ

▍ ルーティング（App Router vs Pages Router）

📁App Router（推奨・Next.js 13+）

app/ ├─ layout.tsx ├─ page.tsx → / ├─ about/ │ └─ page.tsx → /about ├─ blog/[slug]/ │ └─ page.tsx → /blog/:slug └─ api/hello/ └─ route.ts → /api/hello
✅ Server Components デフォルト
✅ Layouts / Loading / Error UI
✅ Streaming / Suspense 対応

📁Pages Router（従来方式）

pages/ ├─ index.tsx → / ├─ about.tsx → /about ├─ blog/ │ └─ [slug].tsx → /blog/:slug └─ api/ └─ hello.ts → /api/hello
✅ getServerSideProps (SSR)
✅ getStaticProps (SSG/ISR)
✅ 安定した実績あり

Image 最適化

🔤
Font 最適化
next/font でレイアウトシフトゼロのフォント読み込み

i18n
