shadcn/ui

コピー&ペーストで使う新世代UIライブラリ

💡 そもそも何が違うのか
従来のUIライブラリ
  • npm installして使う
  • スタイルを上書き困難
  • ライブラリに依存し続ける
  • バージョン管理が必要
shadcn/ui
  • コードをコピーして使う
  • 完全にカスタマイズ可
  • 自分のコードになる
  • 依存関係が最小限
CORE CONCEPT

shadcn/uiは「ライブラリ」ではなくコンポーネント集。
CLIでソースコードをプロジェクトに直接コピーし、
完全に所有・編集できる設計思想が特徴。

🏗 技術スタック構造
あなたのアプリ層
shadcn/ui コンポーネント（あなたのコード）
↑ コピーして配置
Radix UI（アクセシビリティ・動作ロジック）
↑ 土台として利用
Tailwind CSS（スタイリング）
↑ クラス付与
React / Next.js（フレームワーク）
🚀 使い方の流れ
1

プロジェクト初期化

Next.js + Tailwind環境にCLIを実行

npx shadcn@latest init
2

必要なコンポーネントを追加

CLIが src/components/ui/ にコードを生成

npx shadcn@latest add button
3

コードを直接カスタマイズ

生成されたファイルを自由に編集・拡張可能

components/ui/button.tsx を編集
4

インポートして使用

通常のコンポーネントとして import するだけ

import { Button } from "@/components/ui/button"

✅ こんなプロジェクトに最適

・Next.js / Vite + React 環境
・デザインを細かくコントロールしたい
・アクセシビリティを確保したい
・長期的に保守するプロダクト
・独自デザインシステムを構築したい

⚠️ 注意点

・Tailwind CSS の知識が前提となる
・Vue・Svelte等Reactの外は基本非対応
・コンポーネント更新は手動での対応が必要
・大量のコードがプロジェクト内に入る
