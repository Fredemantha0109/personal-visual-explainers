⚡Next.js とは？

Next.js（ネクストジェイエス）

Webサイト・Webアプリを作るための「開発フレームワーク（道具セット）」

フレームワークとは「よく使う機能があらかじめ入った便利な道具セット」のこと。 一から作ると大変な機能（ページ移動、画像最適化など）が最初から使える状態になっています。 Next.jsは特に React（リアクト）※ をベースに作られています。

※ React = 画面のパーツ（ボタン、リストなど）を作るためのJavaScript製ライブラリ（部品集）

🔗 ページ自動生成

「pages」フォルダにファイルを置くだけでURLが決まる。設定不要！

🚀 高速表示

ページを事前に作っておく仕組みで、表示が爆速になる

🔧 サーバー機能内蔵

別でサーバーを用意しなくても、データ処理ができる

🖼 画像最適化

画像を自動で小さく・速くしてくれる機能が標準搭載

🏭「ページをいつ作るか」が選べる（レンダリング方式）

レンダリングとは「HTMLページ（画面）を作ること」。 Next.jsの最大の特徴は、いつ・どこでページを作るかを目的に応じて選べる点です。

📦 SSG（静的生成）
Static Site Generation
ビルド時（公開前の準備段階）にページを全部作っておく方法
例え：試験前に模範解答を用意しておいて、当日はそれを渡すだけ
向き：ブログ・会社HP
⚡ SSR（サーバー生成）
Server Side Rendering
ユーザーがアクセスするたびに、サーバーがその場でページを作る方法
例え：注文が入ってから料理を作るレストラン
向き：ニュース・在庫表示
🔄 ISR（定期更新）
Incremental Static Regeneration
SSGで作ったページを、一定時間ごとに自動で更新する方法
例え：1時間ごとに更新される電光掲示板
向き：ECサイト・価格表
🖥 CSR（ブラウザ生成）
Client Side Rendering
ページの中身をブラウザ側でJavaScriptを使って作る方法
例え：材料を渡して「自分で作ってね」と言う方式
向き：ダッシュボード・管理画面
⚖️React と Next.js の違い

「ReactとNext.jsって何が違うの？」という疑問を解消します。

⚛️ React だけ
🔧画面パーツ（コンポーネント）を作る道具
❌ページ移動の仕組みは自分で用意
❌サーバー機能は別途必要
❌SEO対策（検索に載りやすくする）は難しい
🎯自由度が高い分、設定が多い
▲ Next.js
✅Reactの機能が全部使える
✅ページ移動は自動で設定済み
✅サーバー機能が内蔵されている
✅SEOに強い（SSR/SSGのおかげ）
🎯すぐ開発を始められる
💡 まとめると： ReactはNext.jsの「エンジン」部分。Next.jsはそのエンジンに「カーナビ・エアコン・安全装備」を追加した完成車のようなもの。
📁フォルダ・ファイルの構成

Next.jsのプロジェクトを作ると、こんなフォルダ構成になります。App Router（最新版の方式）の例です。

my-app/ ← プロジェクトの全体フォルダ
├── app/ ← ページやレイアウトを置く場所
│ ├── page.tsx ← トップページ（https://example.com/）
│ ├── layout.tsx ← 全ページ共通のデザイン枠（ヘッダー等）
│ └── about/
│ └── page.tsx ← aboutページ（/about）← フォルダ名がURLになる！
├── public/ ← 画像・ロゴ等の静的ファイル置き場
├── components/ ← ボタンやカードなど再利用する部品
└── package.json ← 使っている道具のリスト（設定ファイル）
