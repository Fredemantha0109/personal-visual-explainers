# コンテンツ設計ガイド

## 推奨情報源（ドメイン別）

Step 1の情報収集では、以下の情報源を **優先的に参照** する。一般的なウェブ検索より先にここに当たる。

| ドメイン | 優先する情報源 | 参照すべき内容 |
|---------|-------------|-------------|
| **会計・監査** | [ASBJ（企業会計基準委員会）](https://www.asb.or.jp/) | 会計基準・適用指針の原文 |
| | [JICPA（日本公認会計士協会）](https://jicpa.or.jp/) | 監査基準・実務指針 |
| | [金融庁](https://www.fsa.go.jp/) | 内部統制報告制度・開示規制の公式解説 |
| | [IFRS財団](https://www.ifrs.org/) | IFRSの原文（英語。日本語訳はASBJ） |
| **法律・規制** | [e-Gov法令検索](https://elaws.e-gov.go.jp/) | 会社法・金融商品取引法等の条文原文 |
| | [金融庁](https://www.fsa.go.jp/) | 金商法・開示規制の解釈・Q&A |
| | [東証（日本取引所グループ）](https://www.jpx.co.jp/) | 上場規程・開示基準 |
| **AI・テック** | 各サービスの公式ドキュメント | OpenAI / Google / Anthropic 等 |
| | [IPA（情報処理推進機構）](https://www.ipa.go.jp/) | セキュリティ・ITガバナンス関連 |
| | [総務省・経産省](https://www.meti.go.jp/) | AI戦略・データ活用に関する公的ガイドライン |
| **経営・マネジメント** | 経産省・内閣府の公表資料 | PMI・コーポレートガバナンス関連 |
| | 学術論文・HBR等 | 戦略フレームワークの原典確認 |

**使い方の原則**:
- 会計・法律のトピックは必ず公式ソースで定義を確認してから図解に落とす
- AI/テックは進化が速いので公式ドキュメントの日付を確認し、最新版を参照する
- 公式ソースのURL（または出典名+条文番号）を「核心ひとこと」作成前にメモしておく

---

## 基本思想

この図解は **「教科書」ではなく「記憶トリガー」** として設計する。

- 一度学んだことを何度も検索し直さないための、個人の知識ストック
- 見返したとき、1分で「そうそう、そういうやつ」と核心が甦る
- 専門家（公認会計士）が見るので、専門用語は噛み砕かなくてよい。ただし **混乱しやすいポイントは明示する**

---

## ① 核心ひとこと — 記憶のトリガーを作る

最も重要なセクション。ここだけ見れば80%思い出せる、という1〜2文。

**良い「核心ひとこと」の条件**:
- 「定義を言い換えた」だけでなく、**本質をついている**
- 読んだ瞬間に頭の中でイメージが動く
- 「なぜ存在するか（Why）」まで含む

**ドメイン別の書き方**:

| ドメイン | 核心ひとことの方向性 |
|---------|------------------|
| AI・テック用語 | 「何を解決するための技術か」＋「従来の何に相当するか」 |
| 会計・監査 | 「誰のために、何を保証/証明するものか」＋「根拠となる基準」 |
| 法律・規制 | 「誰が誰に対して課す義務か」＋「違反時のリスク」 |
| 経営・マネジメント | 「どんな状況で使う概念か」＋「成功の判断基準」 |

## ② 全体構造図 — 位置づけを即把握

トピックの種類に応じて視覚パターンを選ぶ:

| トピックの種類 | 使うパターン |
|-------------|------------|
| **プロセス・手順** | A: 横並びフロー（Step 1 → 2 → 3） |
| **AとBの違い** | B: 左右2カラム比較 |
| **階層・包含関係** | C: 入れ子ブロック（外枠→内側） |
| **数値・規模感** | D: 大きな数字カード（3カラム） |
| **会計基準・法令の条文** | E: 条文引用＋実務解釈対比 ★会計・法律専用 |
| **法改正・新基準の新旧比較** | F: 改正ビフォーアフタータイムライン ★会計・法律専用 |
| **全体の中の位置づけ** | 相関図・マップ（中央に主役、周囲に関連概念） |
| **時系列・変遷（改正以外）** | A: 横並びフロー（縦または横） |

---

## ③ 押さえる3点 — 要点を3つに絞る

必ず3点に絞る（多くても4点）。優先度の高い順に配置。

**絞り方の基準**:
1. 「これを知らないと実務で困る」→ 必ず入れる
2. 「これを誤解していると恥ずかしい」→ 入れる
3. 「あると嬉しいが、なくても困らない」→ 省く

## ④ 実務場面 — 自分の仕事と紐付ける

以下の業務コンテキストの中から、このトピックが実際に登場する場面を2〜4個選ぶ:

| 業務 | 登場しやすいトピック例 |
|-----|---------------------|
| **上場準備** | 開示規制、内部統制報告書、ショートレビュー、有価証券報告書、IFRS適用 |
| **監査法人対応** | 監査基準、重要性の基準値、内部統制、会計処理の見解相違 |
| **内部統制** | J-SOX、COSO、統制の設計・運用評価、ITGCとITAC |
| **PMI / 組織再編** | 企業結合会計、のれん、ステップ取得、共通支配下取引、持分法 |
| **会計アドバイザー** | 会計基準の適用指針、収益認識、リース、金融商品 |
| **管理業務改善** | BPR、KPI設計、業務フロー整備、システム導入効果測定 |
| **組織運営サポート** | 取締役会・株主総会運営、コーポレートガバナンス、役員報酬設計、子会社管理 |
| **AI・システム活用** | AI用語、クラウド契約、データガバナンス、セキュリティ規制 |

## ⑤ 混乱しやすいポイント — 記憶の精度を上げる

何度も確認してしまう原因の多くは「似た概念との混同」か「定義の細部の忘れ」。

**混乱ポイントの書き方**:
- 「× よくある誤解」と「○ 正しい理解」を並べる
- 混乱しやすい類似概念との違いを1行で書く

**よくある混乱パターン**（このドメインで発生しやすいもの）:

| カテゴリ | 混乱しやすいペア例 |
|---------|----------------|
| 会計 | 費用 vs 損失 ／ 引当金 vs 負債 ／ 資本的支出 vs 収益的支出 |
| 監査 | レビュー vs 監査 ／ 内部監査 vs 外部監査 ／ 合理的保証 vs 限定的保証 |
| 法律 | 有報 vs 内部統制報告書 ／ 会社法監査 vs 金商法監査 |
| AI/テック | 機械学習 vs 深層学習 ／ LLM vs AIエージェント ／ APIとMCP |
| PMI | のれん（会計）vs ブランド価値（経営）／ 持分法 vs 連結 |

## ⑥ 関連キーワード — 知識ネットワークを作る

このトピックから芋づる式に辿れる関連概念を5〜10個リスト。

**選び方**:
- 「この単語が出てきたら、このトピックも一緒に思い出したい」という関連性
- 図解はまだない（今後作りたい）概念も含めてよい

## 情報量の調整

**少なすぎる（避ける）**:
- 核心ひとことだけで終わっている
- 実務との紐付けがない
- 混乱ポイントを書いていない

**多すぎる（避ける）**:
- 教科書のような網羅的な解説
- 「当然知っている」前提知識の説明
- 実務に関係しない周辺知識
